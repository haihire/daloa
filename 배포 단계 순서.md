배포 단계 순서
1단계 — 도메인 구입
가비아, 후이즈, Namecheap, AWS->Rount53 등에서 도메인 구입
예: loahub.kr, loaboard.com 등
Vercel 배포 전에 있어야 함
2단계 — EC2 세팅
AWS 계정 생성 + EC2 인스턴스 생성 (t3.small, Ubuntu 22.04)
보안 그룹: 22(SSH), 3001(NestJS), 80, 443 포트 오픈
서버에 설치:
Node.js, npm
MySQL
Redis
PM2 (NestJS 프로세스 관리)
NestJS 빌드 후 PM2로 실행
.env 파일 세팅 (DB 접속정보, API 키 등)
3단계 — Vercel 배포
GitHub에 코드 푸시
Vercel에서 client 폴더를 프로젝트로 연결
환경변수 설정:
도메인 연결
4단계 — HTTPS 설정 (필수)
EC2에 Nginx + Let's Encrypt로 SSL 적용
api.도메인.com → NestJS 3001 리버스 프록시
Vercel 환경변수를 https://api.도메인.com으로 변경
5단계 — 크롤러 자동화
EC2에 Python 환경 세팅
crontab으로 주기적 실행 등록
