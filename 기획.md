서버

- SSR
- SEO 가 가장 중요하다
- 로아 사이트 라고 검색하면 제일 상단에 뜨게 하고 싶다
- 사이트명은 "다로아"
- 검색되면 "다로아 - 로아 사이트 모음" 이렇게 뜨면 좋겠다

Redis

- 자주 가져오는 항목들 미리 대기시켜두면 좋을 것 같아서
- 자주 가져오는 것들이 뭔데?
- 1. 도움되는 사이트들 2. 방송인목록 3. 캐릭터 순위
- 그럴거면 전부 DB에 저장하는게 낫지 않겠니
- 누군가 해당 사이트를 방문했다면 db에 있는 것들을 redis에 저장하고
- 다른 사람이 그 사이에 들어온다면 제한시간 초기화
- 시간이 지나면 다시 db에서 긁어오기

에러 및 예외

- 사이트 변경사항 있으면 카톡 알림
- 에러나도 카톡 알림

주의사항

- GIT 에는 env와 module 는 저장하면 안됨
- ES6 사용은 2026년 하반기 예정
  Decorator 문제: NestJS의 핵심인 데코레이터 메타데이터가 ESM 환경에서 불안정
  의존성 충돌: 많은 NestJS 라이브러리가 CJS 전용이라 ESM 환경에서 import 시 에러가 뿜어져 나옵니다.
  결론: 현재 버전에서 순수 네이티브 ESM으로 NestJS 프로젝트를 구성하는 것은 "기술적으로는 가능하나, 실무적으로는 권장하지 않는 고난의 길"입니다.
