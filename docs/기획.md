# 다로아 기획서

## 프로젝트 개요

- 로스트아크에 도움이 되는 사이트들을 한 곳에 모아 보여주는 허브 사이트
- 사이트명: **다로아**
- 검색 노출 목표: "다로아 - 로아 사이트 모음"
- 핵심 전략: SSR 기반 SEO 최적화 ("다로아", "로아 사이트 모음" 검색 상단 노출)

## 구현된 기능

### 사이트 모음

- 사이트 그리드 표시, favicon 활용

### 특성별 캐릭터 분포

- StatBuildList (치신/신특/치특 등)
- 탭 스크롤 메모리 문제 해결: 탭 전환 시 이전 탭의 스크롤 위치가 유지되는 현상 → `key={activeTab}` 추가로 탭 전환 시 DOM 강제 재생성

### AI 직업 한줄평

- ClassSummaryList (클래스별 한 줄 평)
- 탭 스크롤 메모리 문제 해결: StatBuildList와 동일한 현상 → `key={activeTab}` 적용
- ⚠️ 인벤 크롤링 기반 — 사용 허가 관련 인벤 측에 메일 문의 중 (답변 대기)

### YouTube 인기 영상

- YoutubeList 구현
- 스크롤바 미표시 문제 해결: 스크롤 가능 여부를 사용자가 알 수 없었던 문제 → `rotateX(180deg)` 기법으로 상단에 스크롤바 배치, ResizeObserver로 실제 리스트와 너비 동기화
- 네비게이션 불편 문제 해결: 마우스 드래그·터치만 가능했던 문제 → 헤더 옆 좌/우 버튼(h-10 w-10) 추가, 스크롤바와 버튼 이중 동기화로 UX 일관성 확보

### 즐겨찾기

- 즐겨찾기 기능 부재 문제 해결: 매 방문마다 원하는 사이트를 다시 찾아야 했던 문제 → 별 아이콘 클릭으로 localStorage(`loa_favorites`)에 저장, 새로고침 후에도 유지

- 정렬: 즐겨찾기 상단 고정 (추가 순서)

- Google Analytics 사용 (페이지뷰·기기·지역 수집)

## 아키텍처 결정

### 서버

- NestJS + SSR 기반
- 캐시 전략: 첫 요청 시 DB/API 조회 후 Redis 저장, TTL 만료 전까지 Redis에서 반환
  - 사이트 목록: TTL 10분
  - 캐릭터 순위: TTL 1시간
  - YouTube 영상: TTL 4시간 (EC2 Redis 읽기 전용, Cron 3시간마다 갱신)

### 에러 · 알림

- 모음 사이트 상태 주기적 체크 (매일 오전 9시): 페이지 타이틀 변경 또는 접속 불가(5xx/타임아웃) 감지 시 카카오톡 "나에게 보내기"로 알림
  - 메시지 형식:
    ```
    🔔 로아 사이트 변경 감지 (2026. 5. 5.)

    [KLoa]
      제목 변경: "구제목" → "신제목"

    [인벤]
      ⚠️ 접속 불가 (503)

    [사사게 검색기]
      ✅ 복구됨
    ```
- 카카오 리프레시 토큰 자동 갱신 시 알림 / 만료 D-7 이내 알림
  - 메시지 형식:
    ```
    ✅ 카카오 리프레시 토큰 자동 갱신 완료
    새 만료일: 2026. 7. 4.
    .env 파일 자동 업데이트됨
    ```
    ```
    ⚠️ 카카오 리프레시 토큰 만료 임박 (D-7)
    만료일: 2026. 7. 4.
    자동 갱신 실패 — EC2 .env의 KAKAO_REFRESH_TOKEN 수동 업데이트 필요
    ```
- 서버 에러 발생 시 카카오톡 알림 (401·404 제외, 5xx는 1분·4xx는 5분 쿨다운)
  - 메시지 형식:
    ```
    🚨 [500 InternalServerError]
    경로: GET /api/sites
    메시지: Something went wrong
    ```

## DB 스키마

- **loa_users**: 크롤링된 캐릭터 데이터. `class` → `loa_class.idx` (FK), `core_sun/moon/star` → `loa_ark_grid.core` (코어 종류 참조)
- **loa_class**: 직업 목록 (직업명, 각인명, 성별 등). `loa_users`에서 JOIN해 직업 정보 조회
- **loa_ark_grid**: 아크패시브 코어 목록 (코어명, sun/moon/star 구분, 순서). `loa_users`에서 코어 종류 매핑에 사용
- **loa_sites**: 모음 사이트 목록 (이름, URL, 카테고리, 설명, 아이콘, 상태 체크용 last_title/last_status)
- **loa_class_summaries**: AI 직업 한줄평 데이터 (직업명, 요약 텍스트)

## 제약 사항

### Git
- `.env`, `node_modules` 커밋 금지

### 외부 API 제한
- **로스트아크 Open API**: 분당 100회 한도 → 80회로 자체 제한 (서버에서 rate limiter로 관리)
- **YouTube Data API v3**: 일일 할당량 소진 가능 → 멀티 키 순환으로 대응
- **카카오 리프레시 토큰**: 유효기간 60일(2개월)
  - 만료 30일 전부터 Kakao가 액세스 토큰 갱신 응답에 새 리프레시 토큰 자동 포함 → 서버가 감지 시 `process.env` + EC2 `.env` 파일 자동 교체 후 카톡 알림
  - 매일 오전 9시 크론: `KAKAO_REFRESH_TOKEN_ISSUED_AT` 기준 만료 D-7 이내면 강제 갱신 시도, 실패 시 수동 안내 알림
  - EC2 `.env` 필수 항목: `KAKAO_REFRESH_TOKEN_ISSUED_AT=YYYY-MM-DD` (토큰 발급일)
